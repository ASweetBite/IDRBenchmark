Transformation Name: Branch Inversion (Condition Negation) • Description: Negates the logical condition of an if statement and reorganizes the corresponding execution branches to maintain the original logic. If the original if statement lacks an else branch, the transformation introduces an empty if block for the newly negated condition and relocates the original execution body into a newly generated else block (e.g., if (condition) { execute_code(); } is transformed into if (!(condition)) { /* empty block */ } else { execute_code(); }). If an else branch already exists in the original syntax, the code blocks for the true and false paths are simply swapped.

Transformation Name: Declaration-Initialization Splitting • Description: This transformation locates local variable declarations that are initialized inline on the same line (e.g., Type variable_name = expression;) and separates them into two distinct, sequential statements: a standalone declaration (Type variable_name;) followed immediately by an assignment (variable_name = expression;). Because the variable is still allocated in the same scope and receives the exact same value before any subsequent instructions can read it, the program's control flow, memory state, and observable semantics remain completely unchanged. Every single one of the five provided programs contains inline initializations (such as unsigned int payload_size = ... or unsigned int ia_valid = ...) that can be safely decoupled using this method.

Transformation Name: Temporary Variable Extraction • Description: Identify nested sub-expressions, chained function calls, or complex arithmetic and bitwise operations within the code and extract them by assigning their intermediate results to newly declared, uniquely named temporary local variables. The original complex expressions are then replaced with these temporary variables, which flattens the structural representation of the source code while strictly preserving the exact execution order and logical semantics of the original program.

Transformation Name: Boolean Identity Expansion • Description: This transformation modifies existing control flow conditions (such as those in if statements or loops) by appending a logically neutral tautology or identity. For example, any given conditional check like if (expr) is systematically expanded to if ((expr) && 1) (or && true), or alternatively using an OR identity like if ((expr) || 0). Because evaluating this appended logical identity does not change the truth value of the original condition, the execution paths, variable states, and overall semantics of all the programs are strictly preserved while fundamentally altering the Abstract Syntax Tree (AST).

Transformation Name: Member Access Symmetry Transformation • Description: This transformation replaces the syntax used for accessing members of classes or structures with its logical equivalent. In C and C++, the arrow operator (->) and the dot operator (.) are syntactically distinct but can be interchanged by explicitly applying the address-of (&) or dereference (*) operators. This transformation identifies all instances of pointer->member and converts them to (*pointer).member. Simultaneously, it identifies all instances of object.member and converts them to (&object)->member. This process is applied recursively to nested member accesses. By utilizing the fundamental language identity where E1->E2 is equivalent to (*E1).E2, the transformation preserves the program's logic and side effects while systematically altering the source code's syntactic structure across all expressions.

Transformation Name: Comma-Operator Expression Wrap Description: This transformation wraps an expression $E$ within a comma-separated sequence $((void)0, E)$. In both C and C++, the comma operator evaluates its first operand, discards the result, and then evaluates the second operand, returning its value and type. By using the neutral prefix $(void)0$, the transformation preserves the original program's execution logic, side effects, and values while altering the syntactic structure of the expression. This transformation can be applied to any expression that is not part of a constant-expression context (such as array dimensions or switch case labels). Specifically, it is applied to expressions found in variable initializations, assignments, function call arguments, and return statements. For example, a statement like `x = func(y);` is transformed into `x = ((void)0, func(((void)0, y)));`.

Transformation Name: Variable Address-Dereference Identity • Description: This transformation identifies every instance of a variable or parameter identifier `v` within the source code and replaces it with the parenthesized expression `(*(&v))`. This is a semantic-preserving transformation based on the pointer identity property in C and C++, where applying the address-of operator `&` creates a pointer to the variable, and the subsequent dereference operator `*` resolves that pointer back to the original object. This transformation can be applied to any variable regardless of its type (e.g., primitive, pointer, or struct) and is valid in both l-value contexts (such as the target of an assignment) and r-value contexts (such as in an arithmetic expression or function argument). It maintains the original program logic while introducing redundant but safe memory address operations.

Transformation Name: Opaque Conditional Wrapping • Description: This transformation encapsulates a target block of code—ranging from a single statement to an entire function body—within an if statement whose condition is a tautology (a predicate that is guaranteed to always evaluate to True). For implementation, a sequence of statements is identified and nested within a new if True: block (or an equivalent tautological expression like if 1 == 1:), with the original statements being re-indented to serve as the if block's body. This preserves the original execution logic and semantics while increasing the nesting depth of the code's control flow.

Transformation Name: Conditional Expression Parenthesization • Description: Identify every conditional expression used within control flow structures, specifically the conditions inside `if` statements, `while` loops, and the condition segment of `for` loops. Wrap the entire expression that constitutes the condition in an additional pair of parentheses. For example, a statement such as `if (x > 0)` is transformed into `if ((x > 0))`, and `while (ptr)` is transformed into `while ((ptr))`. In C and C++, parenthesizing an expression does not alter its evaluation, type, or truth-value; it merely reinforces the grouping of the expression. This transformation is semantically neutral and is applicable to any program containing conditional logic.

Transformation Name: Empty Else Block Insertion • Description: Traverse the abstract syntax tree to identify all if and else if conditional statements that lack an explicit else branch. For each of these identified statements, append an empty else { } block. Taking the false branch of the original condition implicitly does nothing; by introducing an empty block, the false path is explicitly defined as a no-operation (no-op). This transformation effectively standardizes the structural exhaustiveness of conditional statements without altering the original execution paths, control flow, or semantic outcomes of the provided programs.
